Libraries!

In unserem Projekt werden drei Libraries eingesetzt. Die Schnittstelle zur Kinect ist in der OpenNI Library implementiert. Zum Erzeugen eines Fensters und zum Laden und Zeichnen von Bildern wird SFML eingesetzt. Das Laden von XML-Dateien wird von TinyXML-2 übernommen.

OpenNI

OpenNI wurde entwickelt, um die Entwicklung von Anwendungen mit 3D Kamera Input zu erleichtern. Durch die OpenNI API können Anwendungen, zum Beispiel Spiele, unabhängig von dem Gerät entwickelt werden, das die Eingabedaten liefert. Das Hauptziel von OpenNI ist, ein Interface für Anwendungen zur Verfügung zu stellen, die ein Natural User Interface zur Eingabe nutzen (Gesten, Körperhaltungen, usw).

Die OpenNI API stellt Generatoren zur Verfügung, mit denen die konkreten Eingabedaten ausgelesen werden können. Beispielsweise wird der Zugriff auf das Tiefenbild der Kinect Kamera durch die Klasse DepthGenerator ermöglicht.

SFML

SFML (Simple and Fast Multimedia Library) stellt ein Interface zu verschiedenen Komponenten des Computers zur Verfügung und erleichtert so die Entwicklung von Spielen und Multimedia Anwendungen. In unserem Projekt kommt Version 1.6 zum Einsatz, da diese Version auf den Computern im Robotiklabor installiert war.

Folgende Aufgaben kann SFML übernehmen:

  • Ein Fenster erstellen, das auf Tastatur und Maus Ereignisse reagiert.
  • Texte und einfache Formen zeichen, zum Beispiel Rechtecke, Kreise.
  • Bilddateien verschiedener Formate laden und zeichnen, unter anderem jpg, png, gif, tga.
  • Sounddateien verschiedener Formate laden und abspielen, beispielsweise ogg, wav.
  • Diverse Systemaufgaben, zum Beispiel das Erstellen von Threads, Zufallszahlen oder Stoppuhren.
  • Nachrichten und Pakete über das Netzwerk verschicken.

In dem Kinect Menü benutzen wir SFML zum Erstellen des Fensters und zum Zeichnen von Bildern, Texten und Rechtecken. Bei dem Hau den Maulwurf Spiel wird zusätzlich das Sound-Modul und die Stoppuhr benutzt.

TinyXML-2

Das Kinect Menü besteht aus mehreren Menüpunkten, die sich aus Bild, Titel und Beschreibung zusammensetzen. Zusätzlich muss das Menü intern wissen, welcher Befehl bei der Auswahl eines Menüpunktes gestartet werden soll. Es war uns wichtig, dass das Menü leicht geändert werden kann, ohne dass man das Projekt neu kompilieren muss. Daher werden die einzelnen Menüpunkte beim Start des Kinect Menüs aus einer XML-Datei eingelesen.

Beispiel einer XML Menü Datei:
Um ein Menü mit zwei Menüpunkten zu erstellen, kann man folgende XML-Datei verwenden:


<MENU>
    <MENUITEM>
        <IMAGE>images/game_logo.png</IMAGE>
        <TITLE>Game title</TITLE>
        <DESCRIPTION>Game description.</DESCRIPTION>
        <COMMAND>./game_command</COMMAND>
    </MENUITEM>

    <!-- place further menu items here, for example: -->
    <MENUITEM>
        <IMAGE>images/logo.png</IMAGE>
        <TITLE>Whac a mole</TITLE>
        <DESCRIPTION>The Whac a mole game.</DESCRIPTION>
        <COMMAND>./hdm</COMMAND>
    </MENUITEM>
</MENU>
